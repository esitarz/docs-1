Introduction

This guide will teach you how a buyer user can create a personal address. Addresses created in this way are only accessible to the buyer user that created them via the /me/addresss endpoints

Prerequisites

  • Create And Assign Users And User Groups
  • Familiarity with Using Impersonation in our Console

1. Impersonate Buyer User

Begin by learning how to Impersonate a Buyer User. The Buyer User you impersonate will need the “MeAddressAdmin” role in order to access the appropriate endpoint.

2. Create a ME Address

API Reference: Create Me Address
POST https://api.ordercloud.io/v1/me/addresses HTTP/1.1
Authentication: Bearer put_access_token_here
Content-Type: application/json


{
  "Shipping": true,
  "Billing": true,
  "CompanyName": "Four51",
  "FirstName": "Sarah",
  "LastName": "Smith",
  "Street1": "110 North 5th St.",
  "Street2": "Suite 300",
  "City": "Minneapolis",
  "State": "MN",
  "Zip": "55403",
  "Country": "US",
  "Phone": "9522940451",
  "AddressName": "Four51",
  "xp": null
}
The billing and shipping values designate whether the addresses can be used as a billing address, shipping address or both.

Conclusion

An authenticated buyer user would make the call in step two in order to make a personal address that only they can view and edit.

Often in a pure B2B scenario individual users may not have access to maintain a personal address book. The creation and delegation of available addresses may be maintained in an administrative context by the seller. Learn more about creating and assigning shared addresses in an admin context in the Create and Assign A Saved Address guide.
